Python & Web design

I'm a dyslexic Python coder. I love Python. I've worked on a few projects with people over the years, but primarily my focus as of late has been on Flask. This website is run by Flask, another website I run is run by Flask. It just works really well. I have written a few programs with Python, namely 'Autobuild' and 'PyBlog'. I have to get the source for those from someone else because I no longer have them. I will update this as soon as I do.

I like Python so much because the syntax is very easy for me to read, especially because I'm dyslexic. I've done other langauges before, such as PHP, but just never really got my attention like Python did. 'Autobuild' was a tool I wrote that builds Funtoo and Gentoo Linux livecd images using mkisofs and mksqaushfs.

'PyBlog' was exactly what you are probably thinking it is. A blogging framework for a selfhosted blog using Flask. It had a login feature, used PostgreSQL for databases, it was all around pretty decent. It then would just shoot out to an index page and you could do one of two things with it. You could run it on a ReverseProxy with Apache to serve the port (if not listening on default 80), you could set it up to work as a WSGI application (smartest way, I think), or you could just run it standalone as your webserver and just have the blog hosted on the default port. But either way, there was minimal configuration. You would run config.py and it would prompt for things like 'Title of your blog', 'Description of your blog', and it would put those into variables and then pyblog.py would be copied to /usr/bin/pyblog, and you could run it and it would ask for what editor you wanted to use (emacs, nano, vi/vim. I really enjoyed writing it, and once I get it back I'm going to add some more features. I never did fully release it, though so I'll have to do that.

Web design is something that I got into a lot of years ago, in fact it's what introduced me to writing code in the first place. My first website looked pretty bad, with the menu just being no styled links, with a black background and green text (like the old terminal computers). I got into Javascript a little bit when I was younger, but I wasn't a huge fan so I didn't use it much, and now I do for other peoples websites. This is the fist time I've had my own websites using any jQuery, but it's just for the dropdown menu. Which is achievable using CSS, but I just wanted something quick.

The first time I designed a website for someone else was a lot of years ago, and it probably wasn't my best work either. I'd like to think I've come a long way since those days.

Web design is something I used to loathe, primarily because I just wasn't too big a fan of writing everything out, but after I started doing it for other people it started growing on me more and more, and now I plan to use it as a way to increase my income since I am unemployed. It's something that is a lot of fun to see how just words can turn into such a cool looking webpage. I have designed my own websites for as long as I can remember, whether my blogs, businesses, you name it. I really enjoy writing code, even though it is difficult for me due to learning disabilities, it's one of the few things in life that calm me down.

Linux Sys Admin & Help out in Funtoo IRC

I'm huge into Linux, and anyone who knows me knows this to be true. Especially Funtoo Linux. It is hands down the only distribution I will use. The reason being is because the amount of power I have over my system is great. I don't have to worry about bloat from packages for things I don't need, and I can customize what I install with the various USE flags. But, you can read more about it here.

I currently maintain 4 Linux servers. This one, another one that's hosting Minecraft and the IRC server, another with Rackspace that's hosting another IRC server with Minecraft as well, another that is used specifically for a private Git repository, and lastly one that is hosting virtual servers and containers (LXC and KVM). All of these servers are currently running Funtoo and I love it.

I am good with a lot of different things as well, from Nginx to Apache2, LXC to OpenVZ, KVM to Xen, automation is my best friend. I write a lot of scripts to just automate things for me. I know databases, sqlite3, PostgreSQL, MySQL, I can get around any Linux server. My dream job is to start my hosting company, I love servers. I just love being a Linux sys/server admin. I also really love virtualization. Containers, VM's, if I can have one host and a bunch of little servers, I am happy. I plan to finish launching my hosting company, Vorlux Hosting soon.
